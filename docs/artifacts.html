This page provides a list of the FHIR artifacts defined as part of this implementation guide.

Structures: Resource Profiles

These define constraints on FHIR resources for systems conforming to this implementation guide

Vaccine Credential Bundle

The bundle of resources that represents the clinical content of a digital vaccination record.
Immune Status Profile

Defines constraints and extensions on the observation resource for the minimal set of data to query and retrieve vaccine credential immune status.
Immunization Profile

Defines a profile representing a vaccination for a vaccine credential Health Card.
Patient Profile

Currently this profile does not modify USCorePatientProfile, and can be removed if USCorePatientProfile is sufficient for our use cases.
Vaccine Reaction Observation Profile

Profile for reporting a reaction to a vaccine.

This profile may not be necessary depending on the use cases for this IG, but it’s included for now because we wanted to have value sets corresponding to all the value sets in the IIS core data elements. In this profile, VaccineCredentialVaccineReactionValueSet includes the IIS adverse reaction codes.

These define sets of codes used by systems conforming to this implementation guide

Body site for vaccine administration value set

Terminology: Code Systems

These define new code systems used by systems conforming to this implementation guide

OtherCode Code System

A code system containing codes that signify a code outside a specified value set has been used, using an ‘Other __, Specify’ approach.
